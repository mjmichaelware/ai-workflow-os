# Self-Building Pump Standard

A pump is a reversible intelligence loop.

state -> event -> registry action -> material effect -> proof -> audit -> knowledge -> better future generation

## Required pumps

1. OS state pump
2. Event pump
3. Action registry pump
4. UI button pump
5. Generated app pump
6. Test proof pump
7. Knowledge archive pump
8. Mistake prevention pump
9. Dependency and library registry pump
10. Import export pump

## Rule

The OS must update itself while fixing apps, and update future generated apps while fixing itself.
Every failure becomes a prevention rule.
No heredoc is allowed for core construction.
