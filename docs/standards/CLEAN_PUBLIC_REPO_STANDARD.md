# Clean Public Repository Standard

A public operator project must look trustworthy.

## Required

- Clean README
- Contributor instructions
- Security policy
- Issue templates
- Pull request template
- CI workflow
- CodeQL workflow
- Dependabot config
- Secret-pattern scan
- Agent compatibility files
- No runtime noise in commits
- No fake testimonials
- No unproven claims

## Commit rule

Every meaningful change should compile, test, prove, commit, and push. If the work is experimental, use a branch and open a pull request.
