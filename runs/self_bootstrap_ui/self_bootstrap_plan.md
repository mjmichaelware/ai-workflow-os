# Self Bootstrap Plan

The app creator can create and improve itself through the same gated workflow it applies to target apps.

Prompt:

Create an app that helps me manage projects from my phone terminal with APIs, CLI tools, browser UI, import/export, and tests.

Stages:
- B0 Declare boundary [auto]: Improve AI Workflow OS only. Do not contaminate target apps.
- B1 Inspect self [auto]: Inspect repo, CLI, dashboard, docs, adapters, and proof scripts.
- B2 Build research graph [auto]: Map market categories, patterns, APIs, CLIs, and app-builder ideas.
- B3 Design next adapters [approval required]: Plan approved web, GitHub, package, model, and local-runtime adapters.
- B4 Generate self patch plan [approval required]: Write an evidence-backed plan for improving this OS.
- B5 Patch self [approval required]: Apply approved improvements to this repo only.
- B6 Proof self [approval required]: Compile, verify CLI, verify UI endpoints, and write proof report.
- B7 Commit self [approval required]: Commit proven self improvements.
